## Architecture Recommendation

I recommend using a Data Lakehouse architecture for the food delivery startup. The company collects different types of data such as GPS logs, customer reviews, payment transactions, and menu images. These include structured, semi-structured, and unstructured data. A Data Lakehouse allows storing all these data types in one place.

The first reason is flexibility. A Data Lakehouse can store structured data like transactions, semi-structured data like JSON logs, and unstructured data like images and text reviews.

The second reason is cost efficiency. Data Lakes are cheaper for storing large volumes of raw data compared to traditional data warehouses.

The third reason is analytics performance. With tools like DuckDB and Spark, the company can run analytical queries directly on the data lake.

Therefore, a Data Lakehouse is the best choice because it provides flexibility, scalability, and efficient analytics in one system.
